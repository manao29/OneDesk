# OneDesk

> 给财税、代账和公司注册服务商的一张数字办公桌。

OneDesk 是一个本地优先的小红书获客 MVP。它用两位固定职责的数字员工，把“内容准备—人工发布—评论整理—线索判断—人工跟进—周复盘”放进一个可追溯的工作台。

![OneDesk 办公室](app/public/assets/onedesk-office-hero-v1.webp)

## 它解决什么问题

- **阿禾｜小红书内容运营专员**：根据公司资料、服务口径、合规禁区与参考笔记，生成可审核的内容草稿、标题、封面文字、标签和发布清单。
- **阿盈｜小红书获客专员**：对用户手工导入的公开评论进行需求分类、意向评分、理由说明和拟回复生成。
- **老板只处理关键动作**：确认内容、人工发布、确认强意向回复、人工跟进并回填结果。

第一版刻意不做自动私信、自动评论、批量互动、账号池或绕过平台风控。所有平台上的外发动作都由用户在小红书内完成。

## 3 分钟启动（新 Mac）

准备：macOS、Node.js 22 或更高版本、Git。

```bash
git clone https://github.com/manao29/OneDesk.git
cd OneDesk/app
npm ci
npm run build
npm start
```

浏览器打开 [http://127.0.0.1:8930/#/](http://127.0.0.1:8930/#/)。

开发模式（同时启动 API 与 Vite 热更新）：

```bash
cd OneDesk/app
npm ci
npm run dev
```

开发预览地址是 [http://127.0.0.1:8931/#/](http://127.0.0.1:8931/#/)。

## 首次使用

1. 打开“知识库”，填写公司与服务的必填资料；完整度达到 60% 后员工才能开工。
2. 到“内容”生成并确认今天的 3 篇内容。
3. 复制发布清单，在小红书 App 内人工发布，再回填帖子链接。
4. 到“线索”粘贴评论、上传 CSV 或单条录入；阿盈会评分、写理由和拟回复。
5. 确认 80 分以上线索的回复，复制文本后在小红书内人工发送，并登记跟进结果。
6. 在“数据”回填发布表现，生成周复盘。

办公室页会根据真实任务状态显示：阻塞、工作中、等待确认、已完成。工作中的“正在输入”动效只在任务真实运行时出现。

## 本地数据与模型配置

- 默认使用 SQLite，本地数据位于 `app/data/`，首次启动会自动创建。
- 未配置模型也能以演示模板和规则引擎跑完整流程。
- 在“设置”中填写 OpenAI 兼容的 Base URL、模型名和 API Key，即可启用真实生成。
- API Key、数据库和导出的 CSV 都不会提交到 Git。

如果要把**已有工作数据**迁移到另一台可信 Mac，请通过安全方式单独复制以下内容，而不是提交到仓库：

```text
app/data/onedesk.sqlite
app/data/workspace/
```

## 验证

```bash
cd OneDesk/app
npm run test:smoke
npm run build
```

烟雾测试会在临时数据库中验证：知识库 → 内容生成/审批/发布 → 评论导入/确认 → 周复盘，不会修改你的日常本地数据。

## 当前边界与下一步

- 当前仅支持用户手工查看并导入的公开评论；没有登录、验证码处理、浏览器采集器或自动化互动。
- 下一阶段优先做用户触发、只读、可中断的“采集当前页面”，而不是扩大到更多行业、更多员工或复杂 CRM。
- 人物手臂逐帧动画需要独立素材层；当前办公室已经用真实任务状态驱动轻动效，不会把静态循环伪装成后台运行。
