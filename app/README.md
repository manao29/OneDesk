# OneDesk MVP

面向财税、代账、公司注册服务商的小红书获客双员工本地工作台。第一阶段只处理用户手工查看并导入的公开信息；不会登录、采集、发布、私信、关注或评论。

## 运行

需要 Node.js 22 或更高版本。

```bash
cd /Users/hankang/Projects/OneDesk/app
npm install
npm run dev
```

打开 `http://127.0.0.1:8931/`。开发服务器会同时启动本地 API；首次运行会在 `data/` 创建 SQLite 数据库和 CSV 工作区。

生产式本地运行：

```bash
npm run build
npm start
```

然后打开 `http://127.0.0.1:8930/`。

## 日常闭环

1. 在“知识库”填完公司与服务必填资料（至少 60%）。
2. 在“内容”录入公开参考笔记（可选），生成并审核 3 篇内容。
3. 复制发布清单，在小红书 App 内人工发布，再登记帖子链接。
4. 在“线索”粘贴评论、上传 CSV 或单条录入；阿盈只做评分、理由和拟回复。
5. 对 80 分以上线索人工确认，复制回复后自行发送，并记录跟进状态。
6. 在“数据”录入真实的发布表现，生成周复盘。

## LLM 与本地数据

- 未配置模型时，应用以演示模板和规则引擎完整走通流程。
- 在“设置”填写 OpenAI 兼容的 Base URL、模型名和 API Key 后使用真实生成；Key 仅存于本机 SQLite 数据库，请勿将 `data/` 提交或共享。
- CSV 导出默认写入 `data/workspace/`；备份这一目录和 `data/onedesk.sqlite` 即可备份本地数据。

## 验证

```bash
npm run test:smoke
npm run build
```

冒烟测试使用隔离的临时数据库，覆盖“补资料 → 内容生成/审批/发布 → 评论导入/确认 → 周复盘”，不会改动日常本地数据。

## 当前边界

- 没有平台登录、验证码处理、浏览器采集器或自动化互动；这些属于后续的可选阶段，且必须在用户本机、人工登录、只读和限速的前提下另行验证。
- 没有多用户、CRM 合同回款、其他平台或其他行业模板。
