# 小红书获客专员MVP PRD

## 1. 产品概述

本小红书获客 MVP 让用户通过本地Agent（workbuddy，或国内其他无需翻墙合规agent） 从小红书公开内容中发现潜在客户，并生成结构化线索表，最后推送到用户手机端，方便用户人工确认后跟进。

本 MVP 不做自动私信，不绕过平台限制，只做线索发现、整理、拟回复生成与移动端提醒。

## 2. 用户功能需求

第一版用户：财税公司老板 / 代账公司销售 / 本地服务类小老板。

他们的问题是：每天需要在小红书找潜在客户，但人工浏览帖子、评论、判断需求、整理话术非常耗时。需要根据帖子内容和回复，整理出最有需求的客户个性化回复和私信。

**本mvp只做小红书，扩展到以后可能是 抖音/微信视频号等平台。目前也只是集中于财税公司一家一行业。针对特定的业务包括：营业执照代办/合规等。**

## 3. 非功能性需求

- 登录的时候可能需要验证码，workbuddy应该自动推送到手机微信上给用户填写后登录。
- 操作发生在客户的电脑上，不能影响其他工作进行，默认后台进行，收集完成表格放在特定的文件区。
- 陌生人私信在网页端无法实现，需要手机端进行，而且只有一条信息机会，每天找10个左右的客户。

## 4. 核心流程

```
用户输入关键词
↓
本地 Agent 打开小红书网页
↓
搜索相关帖子
↓
采集帖子链接、评论、用户名、用户主页链接
↓
AI 判断是否为潜在线索
↓
生成需求分类、意向评分、拟回复内容
↓
导出 CSV
↓
推送到手机端 / WorkBuddy / 企业微信助手
↓
用户人工确认后发送
```

### 4.1 关键词任务创建

用户可以输入：

```
注册公司 东莞/营业执照代办/工商登记/代理记账/税务代办 等
```

系统生成一个采集任务。

### 4.2 本地采集

本地 Agent 需要采集以下字段：

```
platform
keyword
post_title
post_url
comment_user_name
comment_user_id / profile_url
comment_text
detected_need
lead_score
reason
suggested_reply
created_at
```

### 4.3 线索评分

评分范围：0-100

评分规则：

- 80-100：强意向，建议优先联系
- 60-79：中等意向，可进入跟进池
- 40-59：弱意向，只记录
- 0-39：无效线索，不记录

```jsx
注意：有些帖子下面会让扣1或666，需判断帖子内容是否引导有意向客户扣数字，避免误判。
```

### 4.4 CSV 导出

导出的 CSV 至少包含：

```
帖子链接/用户昵称/用户主页/评论内容/需求分类/意向评分/判断理由/拟回复内容
```

保存到电脑的固定工作区（文件夹）

### 4.5 手机推送

第一版只需要支持一种推送方式：

```
将高意向线索推送到 WorkBuddy（连接手机微信） / 企业微信机器人 / 手机端消息
```

推送内容格式：

```
【小红书新线索】
需求：公司注册
评分：86
用户id：xxx
评论：想问下东莞注册公司大概多少钱？
帖子：xxx
建议回复：你好，看到你在问东莞注册公司……
```

## 5. MVP 不做什么

第一版不做：

```
不自动发送私信
不批量关注
不批量评论
不绕过小红书风控
不做账号池
不做多用户权限系统
不做复杂 CRM
不做全平台适配
```

## 6. 验收标准

第一版成功标准不是功能完整，而是：

```
用户每天能获得 20 条可查看、可判断、可跟进的小红书潜在线索。
```

如果用户愿意每天打开手机接收推送查看线索，MVP 就成立。
