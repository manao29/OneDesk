# **小红书内容运营专员 MVP PRD**

**1. 产品概述**

本 MVP 是 OneDesk 的第二个数字员工：**小红书内容运营专员**。

它面向财税公司、代账公司、本地服务商、中小企业老板，帮助用户完成小红书内容运营的基础工作：

发现爆款内容

拆解爆款结构

生成选题库

生成笔记文案

生成标题与封面文字

排版成小红书风格

生成发布清单

辅助用户人工确认后发布

记录发布结果并复盘

第一版不追求全自动运营，而是让用户每天能稳定产出 3-5 篇“可发布、像人写、有业务转化目的”的小红书内容。

---

**2. 用户功能需求**

第一版用户：

财税公司老板

代账公司销售

本地服务类小老板

跨境电商服务商

中小制造业老板

个人知识服务创业者

他们的问题是：

不知道每天发什么

不知道什么内容容易被客户搜索到

不会模仿爆款

写出来太像广告

标题不会起

封面不会排

发完不知道效果好不好

内容和获客线索没有打通

所以内容运营专员要解决的不是“写一篇文章”，而是：

> 每天帮老板生成一批能发布、能沉淀、能带来线索的内容。
> 

---

**3. MVP 定位**

第一版只做小红书。

第一版只聚焦一个行业模板：**财税 / 代账 / 公司注册 / 工商变更 / 税务合规**。

后续再扩展到：

跨境电商

本地生活

制造业外贸

知识付费

招聘服务

活动策划

企业服务

第一版核心目标：

> 让一个完全不懂小红书的老板，每天也能发出 3 篇合格内容。
>

**5. 关键词任务创建**

用户可以输入：

行业：财税服务

城市：东莞

业务：注册公司、代理记账、税务筹划、工商变更、注销公司

目标客户：刚创业的人、小微企业老板、电商个体户、自由职业者

内容风格：专业但不吓人、像真人经验分享、不要硬广

系统生成一个内容任务：

```
任务名称：东莞财税小红书内容运营
平台：小红书
行业：财税服务
目标：获取公司注册 / 代理记账线索
每日产出：3 篇笔记
内容类型：经验科普、避坑提醒、流程讲解、费用说明、案例分享
```

**7. 爆款拆解能力**

每篇参考内容需要拆解成：

标题类型

开头钩子

用户痛点

内容结构

转化方式

评论区需求

适合模仿程度

风险点

示例：

```
标题：东莞注册公司到底要花多少钱？
标题类型：费用型 / 搜索型
用户痛点：怕被乱收费
内容结构：先说结论 → 拆费用 → 讲避坑 → 引导咨询
转化方式：评论区问城市/行业
适合模仿程度：高
风险点：价格不能承诺过死
```

**8. 选题生成**

内容运营专员每天生成 10-20 个选题。

选题分为 5 类：

**1. 搜索型选题**

适合承接精准需求。

```
东莞注册公司需要多少钱？
个体户和有限公司怎么选？
代理记账一年多少钱正常？
新公司没有收入要不要报税？
```

**2. 避坑型选题**

适合提高点击。

```
注册公司前一定要知道的 5 个坑
不要随便找低价代账，后面可能更贵
公司不经营了，不注销会怎样？
```

**3. 流程型选题**

适合建立专业感。

```
东莞注册公司完整流程
营业执照办下来之后还要做什么？
新公司第一个月财税事项清单
```

**4. 案例型选题**

适合增强真实感。

```
一个电商老板注册公司后踩过的坑
客户以为不用报税，结果补了一堆材料
```

**5. 对比型选题**

适合教育客户。

```
个体户 vs 有限公司，创业初期怎么选？
自己记账 vs 找代账，哪个更适合小老板？
```

---

**9. 内容生成**

每个选题至少生成 3 个版本：

专业版

口语版

强钩子版

每篇笔记包含：

```
标题
封面文字
正文
分段排版
标签
评论区引导
私信/咨询引导
发布注意事项
```

示例：

```
标题：
东莞注册公司，到底要花多少钱？

封面文字：
注册公司费用
别只看低价

正文：
很多老板第一次注册公司，最关心的就是费用。

但注册公司这件事，不能只看“代办多少钱”，还要看后面有没有记账、报税、地址、银行开户这些事项。

一般你要重点确认 4 件事：

1. 营业执照代办是否收费
2. 地址是否真实合规
3. 代理记账怎么收费
4. 后续税务申报谁负责

如果有人只跟你说“很便宜”，但不讲后续维护成本，就要多问一句。

适合刚创业的老板先把行业、地址、开票需求说清楚，再判断怎么注册更合适。

评论区引导：
你可以评论“注册公司”，我整理一份新手注册流程给你。
```

**10. 小红书排版能力**

内容运营专员要自动进行小红书风格排版：

短句

多分段

少大段文字

适当使用编号

重点前置

开头 3 秒抓痛点

结尾有互动

标题适合搜索

封面文字不超过 12-16 字

避免过度营销话术

排版规则：

```
每段不超过 2 行
每篇 500-800 字以内
每篇 3-6 个小标题
每篇至少 1 个明确痛点
每篇至少 1 个行动引导
```

---

**11. 标题生成 Skill**

每篇内容生成 10 个标题，分为：

搜索型

痛点型

避坑型

数字型

对比型

本地型

示例：

```
东莞注册公司需要多少钱？
新手注册公司，别只问价格
注册公司前，这 5 件事一定要问清楚

```

标题评分：

```
搜索匹配度：0-100
点击吸引力：0-100
真实可信度：0-100
营销风险：0-100
综合推荐：0-100
```

**13. 发布方式**

第一版支持 3 种方式：

复制到剪贴板

导出为发布清单

推送到手机端人工发布

不建议第一版直接全自动发布。

原因：

小红书发布通常涉及图片选择、账号状态、验证码、风控、人工审美判断。第一版最好让内容运营专员完成 80%，最后 20% 由用户确认。

发布清单格式：

```

标题：
东莞注册公司，到底要花多少钱？

封面文字：
注册公司费用，别只看低价

正文：
……

标签：
#东莞注册公司 #代理记账 #创业 #公司注册 #财税知识

建议发布时间：
晚上 8:00-10:00

发布方式：
复制到手机小红书发布
```

**15. 与获客专员联动**

内容运营专员和获客专员要打通。

内容运营专员发布内容

↓

获客专员监控评论区和相关搜索

↓

发现潜在线索

↓

生成拟回复

↓

推送给老板

↓

老板人工确认跟进

↓

反馈给内容运营专员

↓

优化下一批内容

这就是 OneDesk 的第一个员工协作闭环。

---

**16. 非功能性需求**

操作发生在用户本地电脑。

不影响用户正常办公。

内容草稿保存到固定工作区。

所有发布前内容必须人工确认。

敏感行业内容要提示合规风险。

营销内容要避免虚假承诺。

不能编造客户案例。

不能冒充真实客户评价。

不能诱导违规私信。

不能批量刷赞、刷评、刷收藏。

---

**17. MVP 不做什么**

第一版不做：

不自动批量发布

不自动刷互动

不伪造真人评论

不搬运原文

不洗稿侵权

不承诺流量

不做多账号矩阵

不做账号池

不做私信群发

不做复杂 CRM

不做全平台适配

不做视频剪辑自动化

---

**18. 验收标准**

第一版成功标准不是“内容爆了”，而是：

每天能生成 3 篇可发布笔记

每篇笔记有标题、封面文字、正文、标签、评论引导

用户愿意人工确认并发布

连续 7 天能稳定产出内容

至少有 1-3 篇内容获得真实评论或咨询

内容能沉淀到选题库和复盘表

更准确的 MVP 成功标准：

> 用户不再问“今天发什么”，而是每天打开 OneDesk，就能看到一批可发布的小红书内容草稿。
> 

---

**19. 一句话定义**
小红书内容运营专员，是一个帮中小企业老板持续研究爆款、生成内容、排版笔记、辅助发布并复盘效果的数字员工。
